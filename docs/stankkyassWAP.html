OSI

OSI (open systems interconnection) is conceptual framework used to understand network interactions. developed by ISO (international organization for standardization.)

(1) physical

physical connection between devices. information transferred via bits. hubs, repeaters, modems, cables

(2) data link

transmits packets to host using mac address. doesn't cross boundary of local network. "frame" layer. receivers mac address obtained by placing ARP (address resolution protocol) request through the network (asking how has this [IP] ??) and destination host replies with its MAC. . devices used are switches and bridges.

(3) network

transmission of data across different networks. packet routing (selection of shortest path to transmit the packet). sender/receiver IP placed in header. packet layer. devices used are routers and switches.

(4) transport

responsible for end-to-end delivery of message. uses TCP (transmission control protocol) or UDP (user datagram protocol). TCP provides acknowledgment of successful data transmission and re-transmits data if any errors are found. UDP doesn't have these features.

(5) session

responsible for establishment/management/termination of connections/sessions between two devices. provides authentication and security.

(6) presentation

provides translation. encryption/decryption. compression.

(7) application

displaying the received information to the user

NETWORKING APPLIANCES

router

forwards data packets between computer networks. uses IP address to transmit data. provides connection to internet.

switch

connects multiple devices on network and forwards data using the source/destination MAC & ports. layer 2 (traditional switch, uses MAC). layer 3 (both switch/router)

firewall

networking security device that monitors and filters incoming/outgoing traffic. uses a preset of parameters to allow/deny traffic.

IDS (intrusion detection system)

layer 3. collects and monitors traffic flow data.

IPS (intrusion prevention system)

layer 3. can actively prevent malicious traffic

load balancer

distributes traffic based on distribution conf (such as. round-robin or least # of connections). layer 4.

proxy

acts as intermediary between client and server. anonymity (masks client IP). content filtering (block access to websites, etc. enforce company polices). caching (stores copies requently accessed content to reduce bandwidth consumption).

NAS (network-attached storage)

file storage bank that is shareable between devices via LAN.

SAN (storage area network)

network of high-speed storage devices accessible via a dedicated storage-specific switch. raid arrays.

WAP (wireless access point)

layer 2. connects devices wirelessly to a wired network.

WLC (wireless LAN controller)

devices that manages multiple AP's (access points).single interface for conf. implements security policies. load balancing. monitoring/reporting.

CDN (content delivery network)

stores website content in bank of connected servers in different geographical locations. decreasing time for end users to access web data. CDN caches data, reduces latency and scale quickly to meet demand.

VPN (virtual private network)

creates secure connection between entities over public/insecure network. VPN verifies/establishes the secure connection prior to routing traffic through tunnel.

QoS (quality of service)

prioritizing network traffic. can be programmed to address connectivity issues (such as. dropped packets, delays, jitter, and errors).

TTL (time to live)

time value applied to data. makes sure packets dont get stuck in a forever loop clogging up network. indicates how long receiving entity can store result. hops/time.

CLOUD CONCEPTS

NFV (network functions virtualization)

process by which devices such as routers, switches, firewalls, etc are placed in virutal environment. allows for multiple networking functions to exits on a single device via hypervisor.

VPC (virtual private cloud)

places all networking functions in cloud environment. separated from other tenants in cloud through authentication and encryption.

network security groups

method of filtering network traffic based on predefined set of rules applied to specific group of usrs/network. rules based on parameters such as port #'s, target destinations or protocols.

network security lists

similar to NSG. applied to all vNICs (virtual network interface cards in a subnet. functions similarly to virtual firewall. allowing/denying network traffic based on predefined rules.

cloud/internet gateway

a gateway controls access to set of resources. cloud gateway used to control access between local resources & cloud resources. internet gateway resides between LAN and internet. often provied by ISP (internet service provider). may be controlled in house for larger enterprises.

NAT (network address translation) gateway

NAT is the protocol that allows for multiple private addresses to connect to single public IP. allows for the connection between private & public network.

VPN

used to securely connect to the cloud environment. site-to-site VPN established secure connection between client & cloud over the public internet. once connection is made/verified on both ends a secure tunnel is created and traffic is routed.

direct connect

dedicated physical connection between client & cloud provider.

public

managed/maintained by third party. open for use by the entire internet. pay-per-use. easy to use. highly scalable. least secure.

private

highest level of control for owner. reserves cloud space for sole use of owner. high security and privacy. expensive.

hybrid

mixture of private and public cloud.

SaaS (software as a service)

application. service provider is responsible for the entire cloud infrastructure. user only allowed to use the software/application. least amount of control. email/games/virtual desktop.

PaaS (platform as service)

platform. client controls data and applications. database/web server.

IaaS (infrastructure as a service)

infrastructure. cloud provider only responsible for hardware and virtualization. client responsible for OS and everything on it. VM/servers/load balancers/storage.

scalability

ability to add or subtract blocks of cloud resources on a as-needed basis.

elasticity

dynamically allocate or terminate resources on a as-needed basis.

multitenancy

refers to the numerous clients accessing the same cloud. the individual tenant needs isolation from other resources in the cloud environment.

INTERNET PROTOCOL TYPES

IP is how data is sent/received over the internet at the (3) network layer. IP provides the addressing data for a packet, while the transport protocol used dictates how the data is sent across the network.

ICMP (internet control message)

management protocol used by IP to provide information about a network. used for trace route/ping to test network connectivity.

TCP (transmission control protocol)

establishes connection-oriented session to send data. SYN/ACK three way handshake. provides acknowledgment of successful data transmission and re-transmits data if any errors are found.

UDP (user datagram protocol)

does not verify connection between hosts. does not verify. unreliable. faster than tcp. good for livestreams/VoIP/online gaming.

GRE (generic routing encapsulation)

developed by cisco. tunneling protocol for encapsulating/transporting a variety of network protocols over IP networks. enables packet to cross network it could not normally cross. GRE is what actually creates the tunnel in PPTP (point-to-point tunneling protocol) in creation of VPN network.

IPsec (internet protocol security)

similar to GRE. more secure method for tunneling via IP.

AH (authentication header)

IPsec protocol to create hash upon sending packet. recipient recreate to ensure authentication. both hashes the same, packet is authenticated. designed to authenticate validity of entire packet, not the contents of the packet.

ESP (encapsulating security payload)

the ESP protocol provides confidentiality via symmetric encryption, data integrity via checksums, data origin authentication, anti-replay service and traffic flow confidentiality.

IKE (internet key exchange)

the 2 phases IPsec uses to create secure tunnels. PHASE 1: 2 sites identify each other & negotiate parameters for authentication & encryption methods. PHASE 2: creation of IPsec tunnel.

unicast

one-to-one communication.

multicast

one-to-many communication.

anycast

one-to-one-of-many or one-to-nearest communication. sends to shortest routing distanace. only available w/ IPv6.

broadcast

one-to-all communication.
